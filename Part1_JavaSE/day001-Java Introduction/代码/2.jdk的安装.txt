一. jdk的安装	
	1. 注意 先查看自己电脑的版本  64/32

	2.安装对应的jdk版本

	3. 不用安装 公共的jre  JDK 自带JRE
	
	4. 安装完的目录
	      bin:可以执行 的文件 
		      javac  
			  java
			  
		  src:源代码


二.配置环境变量

   1.为什么配置环境变量？
       
	   一次编译到处运行 
	     
	   1.写源码hello.java
	   
	   2.javac hello.java --> hello.class
	   
	   3.运行  java hello
	   
	   
	   想要让java的命令在任何地方可以访问，配置环境变量
	   
	   C:\Program Files\Java\jdk1.8.0_172\bin
	   
	2.如何配置环境变量
	   
	   注意： 标点符号都是英文半角
	
		win10：
		  在此电脑/我的电脑 上 右键====》 属性 ====》 高级系统设置 ===》环境变量
		  
		   ===》 系统变量内 ===》path  打开 ===》  新建  ===》 javac 等命令所在目录 粘贴进去
		   
		   ===》 一路确定 下去 
		   
		   ===》 关闭之前的dos窗口  再重新打开  再进行测试 javac 
		  
		win7：
			在此电脑/我的电脑 上 右键====》 属性 ====》 高级系统设置 ===》环境变量
		  
		   ===》 系统变量内 ===》path  打开 ===》 javac 等命令所在目录 粘贴进去;系统变量1;系统变量2;....  
		   
		   ===》 一路确定 下去 
		   
		   ===》 关闭之前的dos窗口  再重新打开  再进行测试 javac 
	
三.配置JAVA_HOME + path
    1.为什么需要配置JAVA_HOME ?
	   因为 可能需要使用 bin目录之外的资源 也需要提供一种快捷的访问方式 
	   配置JAVA_HOME
	   
	2.如何配置JAVA_HOME
	
	  在此电脑/我的电脑 上 右键====》 属性 ====》 高级系统设置 ===》环境变量
		  
		   ===》 系统变量内 ===》新建变量==》 变量名： JAVA_HOME 变量值： bin/src 所在的目录
		   C:\Program Files\Java\jdk1.8.0_172
		   
		更改path(如果是win10系统 可能进行更改后 再重启计算机后 变量不生效  如果发生此情况  则 不要再引用 JAVA_HOME)
		
		%JAVA_HOME%\bin(此操作可选)
		
		 ===》 关闭之前的dos窗口  再重新打开  再进行测试 javac
